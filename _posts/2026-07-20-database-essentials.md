> 오늘의 한 문장: **데이터베이스는 서비스의 '기억'이다.** 넣고 꺼내는 코드(SQL)는 AI가 쓰지만, **무엇을 어떤 모양으로 저장할지, 어떤 DB를 고를지, 어떻게 지킬지**는 내가 정한다.
{: .prompt-tip }

## 1. SQL 한 줄 — 거의 영어다

DB에 "필요한 것만 골라 달라"고 말하는 언어가 SQL.

```
SELECT 이름, 가격 FROM 상품 WHERE 분류 = '케이크';
```
→ "상품 표에서(FROM) 분류가 케이크인 것의(WHERE) 이름·가격을 줘(SELECT)."

- SELECT(조회) · INSERT(생성) · UPDATE(수정) · DELETE(삭제) = **CRUD**.
- 외울 필요 없다. SQL은 AI가 쓴다. 나는 **"이 표에서 뭘 꺼내는/넣는 거구나"** 를 알아보면 된다.

## 2. 표로 생각하기 — 관계형

관계형 DB는 결국 **엑셀 같은 표**다.
- 표 하나 = 한 종류(상품 표) / **행(row)** = 한 건(딸기케이크) / **열(column)** = 항목(이름·가격).
- 표끼리는 **고유번호(ID)로 가리켜** 잇는다. 주문 표엔 상품을 통째로 베끼지 않고 "몇 번 상품"이라고만. 이 **"표로 나누고 관계로 잇기"** 가 '관계형(relational)'의 뜻.

## 3. DB는 왜 빠른가 — 색인(Index)

책 뒤 **'찾아보기'** 와 같다. 자주 찾는 항목(이메일·상품명)에 색인을 만들면 수백만 건도 순식간. 없으면 처음부터 다 훑어 느리다. **"검색이 느려요" → AI에게 "자주 찾는 항목에 색인 넣어줘".**

## 4. 계좌이체가 '반쪽'만 되면? — 트랜잭션 · ACID

A가 B에게 송금 = ①A에서 빼기 ②B에 넣기. ① 직후 정전이면 돈이 증발한다. 그래서 여러 동작을 **하나의 묶음(트랜잭션)** 으로: **"전부 되거나, 전부 안 되거나."** 이 성질 포함 네 약속이 **ACID**.
- **A**(원자성) 반쪽 실행 없음 · **C**(일관성) 규칙 어긋난 상태로 안 끝남 · **I**(고립성) 동시에 만져도 안 꼬임 · **D**(지속성) 됐다면 정전에도 남음.

> 실전 하나: 돈·재고·예약처럼 '동시에 두 번' 눌릴 수 있는 곳은 AI에게 **"동시에 두 명이 마지막 하나를 요청하는 상황도 테스트하고 안전하게 처리해줘"**. 단, **ACID는 백업이 아니다** — 실수로 지운 건 안 돌려준다.
{: .prompt-warning }

## 5. 같은 걸 두 번 적지 않기 — 정규화

단골 주소가 바뀌었는데 주문 100건에 주소를 매번 적어뒀다면 100군데를 고쳐야 한다. 그래서 **"같은 정보는 한 곳에만, 나머지는 가리키게"** — 회원 정보는 회원 표에 한 번, 주문 표엔 "몇 번 회원"만. 이 중복 제거 정리정돈이 **정규화(normalization)**. 나는 데이터 모양을 그릴 때 **"이 정보, 여기저기 똑같이 적히고 있진 않나?"** 만 의심하면 된다.

## 6. 구조를 바꿔야 할 때 — 마이그레이션

데이터가 든 DB의 구조(칸 추가·표 분리)를 안전하게 바꾸는 작업이 **마이그레이션**. "DB 구조 변경을 Git처럼 이력으로 관리(되돌리기 가능)"라 생각하면 된다. **딱 하나: 칸을 삭제하는 변경은 데이터가 영구히 사라질 수 있어, 바꾸기 전 반드시 백업.**

## 7. 캐시와 Redis — 로그인이 빠른 비결

세션 확인은 거의 모든 요청마다 일어나 '빠름'이 생명이다. 그래서 **메모리 기반·키-값**인 **Redis**가 딱 — 조회가 빠르고 **자동 만료(TTL)** 로 "30분 미활동 시 자동 로그아웃"을 쉽게 만든다. "세션을 여러 서버가 나눠 쓰기"와 "캐시로 속도 얻기"가 Redis 하나로 만난다.

## 8. 개인정보(PII) — 덜 모으는 게 가장 안전

이름·이메일·비밀번호가 쌓이면 **자산이자 책임**이다. 네 가지만:
1. **최소 수집** — 목적에 필요한 최소만. "이메일만 되는데 전화·생년월일까지 받고 있진 않나?" (개인정보보호법 제3·16조)
2. **비밀번호는 절대 평문 금지** — bcrypt·Argon2 같은 **비밀번호 전용 해시(+솔트)** 로. (야후 보이스 2012 평문 저장 사고)
3. **민감정보(주민번호·카드)는 직접 저장 X** — 결제는 PG사, 본인확인은 인증 서비스에 위탁.
4. **목적 제한** — 밝힌 목적에만. 3자 제공 시 5가지(받는 자·목적·항목·기간·거부권) 고지 + 동의.

> **가장 안전한 데이터는 애초에 안 가진 데이터.** 모으지 않은 정보는 샐 수도 없다.
{: .prompt-tip }

## 9. 내 컴퓨터에 무료로 — SQLite · PostgreSQL · Docker

- **SQLite** — 설치·서버 없음. **DB 전체가 파일 하나.** 작은 앱·프로토타입에 딱(파이썬 내장, 사실상 설치 없이).
- **PostgreSQL · MySQL** — 회사들이 쓰는 그 오픈소스 DB를 내 컴퓨터에 무료로.
- **Docker** — 설치가 번거로우면 명령 한 줄로 '컨테이너'로 띄우고, 필요 없으면 통째로 지운다.

## 10. 데이터를 잃지 않기 — 백업 3-2-1

해커가 없어도 데이터는 **실수 한 번**으로 사라진다.
- 픽사 「토이 스토리 2」(1998 제작) — 잘못된 명령으로 파일이 지워지는데 백업도 고장. 재택근무자 집 컴퓨터의 2주 전 사본이 영화를 살림.
- GitLab(2017) — 실수로 삭제, 여러 겹 백업이 모두 조용히 고장 → 6시간치 유실.

**CISA 권고 3-2-1 규칙**: 사본 **3벌** · 서로 다른 **2종** 매체 · **1벌은 다른 장소(off-site)**.
그리고 **"복구가 진짜 되는지" 가끔 확인** — 열어본 적 없는 백업은 백업이 아니다. (무료 요금제는 자동 백업이 없는 경우가 많으니 직접 챙길 것.)

## 11. SQL 주입 방어 — "레시피와 재료를 섞지 마라"

사용자 입력을 명령문에 **그대로 이어붙이면**, 입력 자리에 명령을 끼워 넣는 공격(**SQL 주입**)이 통한다. 방어는 **파라미터화 쿼리**: 명령문엔 빈칸(`?`)만 두고 값은 따로 건넨다 → DB가 입력을 **무조건 '그냥 값'으로만** 취급.
- 이건 예전에 배운 **"프론트는 못 믿는다"** 의 DB 버전. 바깥 값은 의심부터.
- AI에게: **"DB에 넣는 값은 문자열로 이어붙이지 말고 파라미터화 쿼리로. 사용자 입력을 그대로 신뢰하지 마."**

---

한 줄 정리: **DB는 서비스의 기억이고, 코드는 AI가 채운다.** 내 몫은 *무엇을 어떤 모양으로 담고(정규화), 어떻게 빨리 찾고(색인·캐시), 안 꼬이게 하고(트랜잭션), 덜 모으고(개인정보), 잃지 않고(백업), 안 뚫리게(파라미터화)* 를 **판단**하는 것. 오늘도 한 걸음.
