나중에 다시 볼 사전이라, **정의는 정확하게 · 예시는 쉽게** 적는다. 각 항목은 정의 → 비유 → 예시.

## 1. HTML / XML
- **정의**: 둘 다 태그(`< >`)로 구조를 표시하는 마크업 언어다. **HTML**은 웹 브라우저에 **화면(문서)을 표시**하기 위한 언어, **XML**은 데이터를 **저장·전송**하기 위해 구조화하는 범용 언어다.
- **비유**: HTML은 사람이 읽는 *신문 지면*, XML은 기계끼리 주고받는 *내용물 명세서*.
- **예시**: 웹페이지는 HTML로, 설정 파일이나 시스템 간 데이터 교환은 XML로.

## 2. CRUD
- **정의**: 데이터 저장소에서 이뤄지는 **네 가지 기본 처리** — 생성(Create)·조회(Read)·수정(Update)·삭제(Delete)의 머리글자.
- **비유**: 수첩에 하는 일 전부 — 적고, 들춰 보고, 고치고, 지운다.
- **예시**: 이 블로그도 CRUD다. 글 작성(C)·방문자 읽기(R)·수정(U)·삭제(D).

## 3. 하드코딩(Hard-coding)
- **정의**: 바뀔 수 있는 값을 설정·변수로 분리하지 않고 **소스 코드 안에 직접 고정해 넣는** 것.
- **비유**: 못을 벽에 그대로 박는 것 — 옮기려면 벽을 뜯어야 한다.
- **예시**: 할인율을 코드에 `10`으로 박으면 값이 바뀔 때마다 코드를 고쳐야 한다. 그래서 값은 한곳에 모아 둔다.

## 4. VS Code / Cursor
- **정의**: 둘 다 소스 코드 편집기다. **VS Code**는 마이크로소프트의 범용 편집기, **Cursor**는 VS Code를 기반으로 **AI 코드 생성·수정 기능을 내장**한 편집기다.
- **비유**: 같은 공구함인데, Cursor는 옆에 *거드는 조수*가 상주하는 쪽.
- **예시**: 편집기 안에서 "이 함수 고쳐줘"를 바로 시키고 싶을 때 Cursor 계열.

## 5. REST
- **정의**: 웹 API 설계 방식의 하나. **자원을 URL로 식별하고, HTTP 동사(GET·POST·PUT·DELETE)로 그 자원을 다루는** 규칙이다.
- **비유**: 창구에서 "몇 번 자원을, 이렇게 처리해 주세요"라고 정해진 양식으로 요청하는 것.
- **예시**: 목록 조회는 GET, 새 항목 추가는 POST — CRUD와 그대로 짝이 맞는다.

## 6. 프론트엔드 / 백엔드
- **정의**: **프론트엔드**는 사용자의 브라우저에서 실행되어 **화면·상호작용**을 담당하는 쪽, **백엔드**는 서버에서 **데이터 저장·연산·권한 처리**를 담당하는 쪽.
- **비유**: 가게의 *매장(프론트)* 과 *주방·창고(백엔드)*.
- **예시**: 신청 화면은 프론트, "정원이 찼는지" 판단은 백엔드 몫.

## 7. 저장 계층 (메모리 vs 디스크·DB)
- **정의**: 데이터를 보관하는 위치의 층. **메모리(RAM)** 는 빠르지만 전원이 꺼지면 사라지고, **디스크·데이터베이스**는 느리지만 영구적으로 남는다.
- **비유**: 지금 펼쳐 둔 *책상 위(메모리)* 와 오래 넣어 두는 *서랍(디스크·DB)*.
- **예시**: 서비스의 최종 보관은 보통 데이터베이스에서 이뤄진다.

## 8. PRD(Product Requirements Document, 제품 요구사항 문서)
- **정의**: 개발 착수 전에 **무엇을·누구를 위해·어떤 규칙으로** 만들지 정의해 두는 요구사항 문서.
- **비유**: 요리 전 레시피, 시공 전 도면.
- **예시**: AI에게 시킬 때 이게 없으면 대화가 길어질수록 방향이 샌다. 목표를 먼저 글로 고정한다.

## 9. HTTP(HyperText Transfer Protocol)
- **정의**: 웹에서 클라이언트(브라우저)와 서버가 **요청·응답을 주고받기 위한 통신 규약**.
- **비유**: 주고받을 때 지키는 *택배 표준 절차*.
- **예시**: 주소창에 주소를 넣으면 브라우저가 "이 페이지 주세요"를 HTTP 요청으로 보낸다.

## 10. API(Application Programming Interface)
- **정의**: 서로 다른 프로그램이 **정해진 방식으로 요청·응답을 주고받도록 규정한 접점(창구)**.
- **비유**: 주문을 받아 주방에 넘기는 *종업원* — 정해진 메뉴로만 부탁한다.
- **예시**: 날씨 API에 지역을 건네면 그 지역 날씨 데이터가 돌아온다.

## 11. JavaScript / TypeScript
- **정의**: **JavaScript**는 웹 브라우저에서 실행되는 대표 프로그래밍 언어. **TypeScript**는 여기에 **타입(값의 종류) 검사를 더해**, 실행 전에 오류를 잡아 주는 언어(컴파일하면 JavaScript가 된다).
- **비유**: 다 풀고 채점하느냐(JS) vs 푸는 중에 빨간 줄로 알려 주느냐(TS).
- **예시**: 숫자 자리에 글자를 넣으면 TS는 그 자리에서 바로 짚어 준다.

## 12. 컴파일(Compile)
- **정의**: 사람이 쓴 소스 코드를 **컴퓨터가 실행할 수 있는 형태(기계어 또는 다른 코드)로 변환**하는 과정.
- **비유**: 원고를 다른 언어로 옮기는 번역.
- **예시**: TypeScript는 실행 전에 JavaScript로 컴파일된다.

한 줄 정리: 오늘 용어는 **데이터를 어떻게 다루고(CRUD·REST·API), 어디서 처리하며(프론트·백엔드·저장), 코드를 어떻게 변환·점검하는가(컴파일·TS)** 로 모인다.
