오늘은 "코드를 외우는 게 아니라 **설계하고 검증하는 힘**을 기른다"는 말이
무슨 뜻인지 몸으로 이해한 날이었다.

## 1. 코딩의 스펙트럼

코딩이라고 다 같은 코딩이 아니었다. 관여하는 정도에 따라 이렇게 나뉜다.

- **논코드(No-code)**: 코드 없이 도구로 만든다
- **로우코드(Low-code)**: 최소한의 코드만 쓴다
- **바이브 코딩(Vibe coding)**: AI와 대화하며 만든다
- **트래디셔널 코딩(Traditional)**: 직접 다 짠다

핵심은, 내가 모든 문법을 외울 필요는 없다는 것.
대신 **무엇을 만들지 설계하고, 나온 결과가 맞는지 검증하는 힘**이 중요하다.

## 2. 웹앱의 3층 구조

검색엔진, 구글, 네이버 지도 같은 웹앱은 결국 세 부분으로 나뉜다.

| 구분 | 역할 | 식당 비유 |
|---|---|---|
| 프론트엔드 | 사용자가 보는 화면 | 홀 |
| 백엔드 | 요청을 처리하는 서버 | 주방 |
| 데이터베이스 | 정보를 저장하는 곳 | 냉장고 |

손님이 홀에서 주문하면 → 주방이 요리하고 → 냉장고에서 재료를 꺼낸다.
웹도 똑같은 흐름이다.

## 3. 화면을 다루는 세 가지 방식

- **CLI**: 명령줄 (글자로 명령)
- **TUI**: 터미널 UI
- **GUI**: 그래픽 UI (마우스로 클릭하는 화면)

## 4. 개발 환경 점검

터미널에서 아래를 하나씩 확인했다. 버전 번호가 나오면 정상 설치된 것.

- `pwd` — 지금 내가 어느 폴더에 있는지
- `node --version`, `npm --version`
- `uv --version`, `git --version`
- `claude --version` → **2.1.207** 설치 확인 ✅

> Claude Code는 Node.js 없이 네이티브 빌드로 `C:\Users\nike1\.local\bin\claude.exe`에 설치됐다.
> `.local` 앞의 점(`.`)은 리눅스에서 **숨김 파일**을 뜻한다.

## 5. 오늘의 핵심 — Claude Code는 "특별한 존재"가 아니다

가장 크게 깨달은 부분. **Claude Code는 터미널도, 셸도, 커널도 아니고
셸 위에서 실행되는 하나의 프로그램일 뿐이다.** 내가 `python`을 실행하는 것과 똑같은 층에 있다.

내가 `claude`라고 치면:

1. 셸이 `$PATH`(맛집 순서표)를 앞에서부터 뒤져 `claude` 실행 파일을 찾고
2. 찾으면 새 프로세스를 띄워 실행하고
3. 그 프로그램이 파일을 읽고, 명령 실행을 요청하고, 결과를 되돌려 준다

비유하면 Claude Code는 **"나를 대신해 주방(운영체제)에 주문을 넣는 종업원"**이다.
종업원이 아무리 유능해도, 주방의 규칙(권한)과 지금 서 있는 홀(현재 폴더)을 벗어날 순 없다.

## 6. 오류를 "층으로 나눠" 진단하기

이게 오늘 가장 실용적이었다. 오류 메시지만 봐도 어느 층 문제인지 좁혀진다.

| 오류 증상 | 의심할 층 | 확인 방법 |
|---|---|---|
| `command not found` | 셸 / PATH | 설치됐는지, `$PATH`에 경로가 있는지 (`type claude`) |
| `Permission denied` | 운영체제 권한 | 파일·폴더 권한 (`ls -l`) |
| 글자 깨짐 · 창 멈춤 | 터미널 · PTY · 인코딩 | 터미널 앱, UTF-8 인코딩 |

- `command not found` = 존재의 문제가 아니라 "순서표에 주소가 없다"
- `Permission denied` = 찾긴 찾았는데 권한이 막은 것 (AI 문제가 아님)
- 글자 깨짐 = 맨 아래 로직이 아니라 맨 위 터미널 층 문제

## 7. AI에게 도움을 요청하는 좋은 습관

AI가 강력해도 **권한과 현재 폴더를 초월하지는 못한다.**
그래서 문제를 물을 때는 세 가지를 함께 주면 좋다.

1. **오류 메시지 한 줄** → 어느 층 문제인지 출발점
2. **`pwd`** → 지금 어느 폴더에 서 있는지
3. **`echo $SHELL` + 실행한 명령** → 어떤 환경에서 무엇을 시도했는지

"그냥 안 돼요"가 아니라 **"이 폴더에서, 이 명령을 쳤더니, 이 오류가 났다"**.
추측을 없애고 재현 가능하게 만드는 방법이다.

## 한 문장 정리

> Claude Code는 셸 위에 얹힌 종업원일 뿐이고, 그 아래로 셸·PTY·운영체제 권한이라는 층이 항상 깔려 있다.
> 문제가 생기면 "AI가 이상하다"가 아니라 **"어느 층에서 막혔나"**를 먼저 보는 게 정확한 진단법이다.
