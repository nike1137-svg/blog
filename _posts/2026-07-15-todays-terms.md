오늘 새로 알게 된 용어들을 **이해·암기용**으로 정리한다. 각 용어를 "정의 + 비유 + 예시"로 짧게.

## 1. HTML과 XML의 차이
- **정의**: HTML은 **화면을 보여주는** 문서, XML은 **데이터를 저장·전달하는** 문서.
- **비유**: HTML은 사람이 읽으라고 만든 *완성된 신문 지면*, XML은 기계끼리 주고받는 *상자 속 내용물 명세서*.
- **예시**: 웹페이지는 HTML, 프로그램 간 데이터 교환·설정 파일은 XML.

## 2. CRUD
- **정의**: 데이터를 다루는 기본 4작업 — **C**reate(생성)·**R**ead(읽기)·**U**pdate(수정)·**D**elete(삭제).
- **예시**: 블로그 글쓰기(C) / 방문자가 읽기(R) / 글 수정(U) / 글 삭제(D). 내 블로그가 곧 CRUD의 예시다.

## 3. 하드코딩(Hard-coding)
- **정의**: 값을 설정·변수로 빼지 않고, 결과물/코드에 **사람이 직접 박아 넣는 것**.
- **비유**: 벽에 못을 그냥 박아 고정 — 바꾸려면 벽을 뜯어야 함.
- **예시**: 할인율 `10%`를 코드에 숫자 그대로 적으면, 바뀔 때마다 코드를 고쳐야 한다.

## 4. VS Code vs Cursor
- **정의**: 둘 다 코드 편집기. **VS Code**는 마이크로소프트의 대표 에디터, **Cursor**는 VS Code를 기반으로 AI 기능(코드 생성·수정·대화)을 깊게 넣은 AI-우선 에디터.
- **비유**: VS Code는 잘 만든 공구세트, Cursor는 거기에 *AI 조수를 상주*시킨 버전.

## 5. REST
- **정의**: 웹에서 데이터를 주고받는 대표 설계 규칙. URL로 자원을 지정하고 HTTP 동사(GET/POST/PUT/DELETE)로 조작.
- **비유**: 식당 주문 규칙 — 메뉴(URL)를 정하고 동작(주문·변경·취소)을 말한다.

## 6. 프론트엔드(Frontend)
- **정의**: 사용자가 브라우저에서 **직접 보고 누르는 쪽** (화면·버튼·입력창).
- **비유**: 식당의 **홀·메뉴판**.

## 7. 백엔드(Backend)
- **정의**: 화면 뒤에서 **처리하는 쪽** (서버·데이터 저장·권한 검사).
- **비유**: 식당의 **주방·창고**.

## 8. 저장(데이터) 계층 — 저장장치 구성
- **정의**: 데이터를 **어디에 어떻게 보관**하는지의 구성. 빠르지만 꺼지면 사라지는 메모리(RAM)와, 느리지만 영구적인 디스크/데이터베이스로 나뉜다.
- **비유**: 지금 쓰는 *작업 책상*(메모리) vs 오래 보관하는 *서랍·창고*(디스크·DB).
- **예시**: 웹서비스의 최종 저장소는 보통 데이터베이스.

## 9. PRD (Product Requirements Document)
- **정의**: 만들기 전에 **"무엇을, 누구를 위해, 어떤 규칙으로"** 만들지 글로 못 박는 제품 요구사항 문서.
- **비유**: 요리 전 *레시피*, 집 짓기 전 *설계도*.
- **예시**: 바이브 코딩에서 특히 중요 — 목표가 글로 고정돼야 AI가 방향을 안 잃는다.

## 10. HTTP (HyperText Transfer Protocol)
- **정의**: 브라우저와 서버가 웹에서 정보를 주고받는 **통신 약속(규약)**.
- **비유**: 택배를 주고받는 *표준 절차*.
- **예시**: 주소창에 URL을 넣으면 브라우저가 HTTP로 서버에 "이 페이지 줘"라고 요청한다.

## 11. API (Application Programming Interface)
- **정의**: 프로그램끼리 정해진 방식으로 **요청·응답을 주고받는 창구**.
- **비유**: 식당 *종업원* — 손님(내 프로그램)과 주방(다른 시스템) 사이에서 정해진 메뉴(요청)로 소통.
- **예시**: 날씨 API에 지역을 보내면 그 지역 날씨 데이터가 돌아온다.

## 12. JavaScript vs TypeScript
- **정의**: JS는 **실행해봐야** 오류를 아는 경우가 많음(런타임). TS는 JS에 **타입**을 붙여, **실행 전에**(작성·컴파일 시점) 오류 위치를 미리 잡아줌.
- **비유**: JS는 시험지 다 풀고 채점, TS는 *푸는 중에 빨간 줄로* 틀린 곳 표시.

## 13. 컴파일(Compile)
- **정의**: 사람이 쓴 **소스 코드**를 컴퓨터가 실행할 수 있는 **기계어(또는 중간 코드)로 번역**하는 과정.
- **비유**: 한국어 원고를 기계가 읽을 언어로 *번역*.
- **예시**: TypeScript → JavaScript로 컴파일된다.
