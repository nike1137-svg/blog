오늘은 에이전트를 실제로 강하게 부리는 데 등장한 용어들을 정리한다. 규격·도구·모델이 섞여 있어서, 손에 잡히는 비유로 하나씩 꽂아 둔다.

## 1. MCP(Model Context Protocol)
- **정의**: AI를 외부 도구·데이터에 연결하는 **표준 규격**. Anthropic이 2024년 말 공개했고 지금은 업계 표준.
- **비유**: **AI를 위한 USB-C 포트** — 한 번 규격에 맞춰 만들어 두면 MCP를 지원하는 아무 AI에나 꽂힌다.
- **예시**: Figma·Notion·DB를 연결해 "우리 DB 보고 답해 줘". 공식 서버 목록은 [MCP 레지스트리](https://registry.modelcontextprotocol.io/)에서 찾는다. 단, 플러그 모양이 같다고 권한까지 같은 건 아니다 — MCP는 연결 규칙이지 자동 안전 보장이 아니다.

## 2. CLI(Command-Line Interface)
- **정의**: 마우스 대신 **텍스트 명령어**를 입력해 컴퓨터·프로그램을 제어하는 방식.
- **비유**: 메뉴판을 손가락으로 고르는 대신(GUI), 주방에 대고 **말로 주문을 외치는** 것.
- **예시**: `gh`로 깃허브 이슈 가져오기. GUI보다 빠르고 자동화·원격 서버 관리에 유리하다.

## 3. bash tool
- **정의**: 에이전트가 **터미널에 명령어를 실행**하게 해 주는 도구.
- **비유**: AI에게 쥐여 준 **키보드 손** — 말만 하는 게 아니라 직접 명령을 친다.
- **예시**: 파일 목록 보기, 테스트 돌리기. 쓰기·삭제처럼 **되돌리기 어려운 명령**은 승인 과정을 둔다.

## 4. PATH
- **정의**: 명령어를 쳤을 때 OS가 **실행 파일을 찾아보는 폴더 목록**.
- **비유**: "이 이름 부르면 이 서랍들 순서대로 뒤져봐"라고 정해 둔 **탐색 경로**.
- **예시**: `python`이라고만 쳐도 실행되는 건 PATH에 등록돼 있어서다. 새 도구가 `command not found`면 PATH부터 의심.

## 5. system prompt
- **정의**: 모델에게 **처음부터 상시로 거는 지시문** — 역할·규칙·톤을 정한다.
- **비유**: 배우에게 미리 건네는 **배역 설정지** — 매 장면 다시 설명하지 않아도 된다.
- **예시**: "너는 정중한 고객상담원이다" 같은 밑바탕. 대화 내내 유지된다.

## 6. 스킬(Skill)
- **정의**: 자주 하는 작업을 한 번 적어 두고 꺼내 쓰는 **재사용 설명서(SKILL.md)**.
- **비유**: 반복 작업의 **플레이북** — "말투 스킬 써서 다듬어 줘" 한마디면 발동.
- **예시**: 평소엔 이름·설명(수십 토큰)만 훑다가 필요할 때만 본문을 읽는다(점진 공개). 그래서 `description`을 잘 쓰는 게 핵심 — AI가 언제 꺼낼지를 그 한 줄로 판단한다. (오늘 내 블로그용 스킬 7개를 직접 만들었다.)

## 7. 훅(Hook)
- **정의**: 특정 시점(저장 직전·커밋 전 등)에 **자동으로 끼어드는 동작**.
- **비유**: 문에 단 **자동 센서** — 지나갈 때마다 정해 둔 일이 알아서 실행된다.
- **예시**: 커밋 전 자동 포맷팅·린트 검사. 매번 시키지 않아도 그 순간에 돈다.

## 8. 플러그인(Plugins)
- **정의**: 스킬·도구·명령을 **묶음으로 통째 배포·공유**하는 단위(Claude 플러그인).
- **비유**: 앱 하나 깔면 기능이 여럿 딸려오는 **패키지**.
- **예시**: 코드 리뷰를 여러 전문 에이전트가 병렬로 나눠 보는 플러그인.

## 9. LSP(Language Server Protocol)
- **정의**: 편집기와 언어 분석기를 잇는 **표준 규격** — 자동완성·오류표시·정의로 이동을 제공.
- **비유**: 편집기용 **MCP** — 언어마다 따로 만들지 않고 한 규격으로 붙인다.
- **예시**: 함수 위에 마우스를 대면 뜨는 타입 정보, 오타 밑의 빨간 밑줄.

## 10. AGENTS.md
- **정의**: 프로젝트의 **상시 안내문** — AI가 항상 참고할 규칙·컨벤션을 적어 둔 파일. 2025년 여러 회사가 공통 표준으로 채택.
- **비유**: 신입에게 건네는 **사내 규정집** — 매번 설명하지 않아도 이걸 먼저 읽는다.
- **예시**: "모든 규칙은 CLAUDE.md에 있으니 먼저 읽어라" 식 포인터. 짧고 명령어 중심으로 쓴다.

## 11. Kimi K3
- **정의**: Moonshot AI가 2026년 7월 공개한 **2.8조 파라미터 오픈웨이트 모델**(역대 최대급, 3T급 최초).
- **비유**: 미국 최상위 모델을 바짝 뒤쫓는 **중국발 추격자**.
- **예시**: 100만 토큰 컨텍스트·네이티브 비전. Opus 4.8·GPT 5.5는 앞섰지만 Claude Fable 5·GPT 5.6엔 아직 뒤. 출력 **$15/M 토큰**.

## 12. 아레나 AI(WebDev / Code Arena)
- **정의**: 두 모델의 결과를 사람이 **투표로 겨루게** 해 순위를 매기는 리더보드(LMArena). WebDev는 프론트엔드 코딩 특화.
- **비유**: 모델들의 **블라인드 시식 대결** — 누가 만든지 모른 채 더 나은 쪽에 표를 준다.
- **예시**: "이 웹 화면 만들어 줘"를 두 모델에 시키고 승자에 투표. 점수는 Bradley-Terry 방식으로 산출.

## 13. 요금 비교(Fable 5 vs Kimi K3)
- **정의**: 같은 출력 100만 토큰 기준, **모델별 가격 차이**.
- **비유**: 같은 거리 택시비 비교 — 브랜드마다 미터기가 다르다.
- **예시**: Fable 5 출력 **$50/M** vs Kimi K3 **$15/M**. 성능은 Fable이 앞서지만 비용은 K3가 약 1/3 — 일의 성격에 맞춰 고른다.

## 14. goose · Pi
- **정의**: **오픈소스 터미널 에이전트**. 무료이고, 쓰는 모델을 골라 끼울 수 있다.
- **비유**: 내가 직접 뜯어고칠 수 있는 **DIY 연장** 에이전트.
- **예시**: goose는 MCP·확장이 풍부(리눅스재단 AAIF), Pi는 미니멀(OpenClaw의 엔진). 오픈소스 에이전트를 직접 손보고 싶을 때.

---

한 줄로 남기면 — 규격(MCP·LSP)은 **한 번 만들어 여러 곳에 꽂는 것**이고, 스킬·훅·AGENTS.md는 **반복 설명을 파일로 굳혀 두는 것**이다. 도구 이름은 바뀌어도 이 뼈대는 남는다.
