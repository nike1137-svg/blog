오늘 데이터베이스를 배우며 만난 용어들을 **이해·암기용**으로 정리한다. "정의 + 비유 + 예시"로 짧게.

## 1. 캐시(Cache)
- **정의**: 자주 쓰는 데이터를 **메모리에 잠깐** 두어 빠르게 꺼내 쓰는 것.
- **비유**: 매번 창고(디스크·DB)까지 안 가고, 자주 찾는 물건을 **계산대 옆 바구니**에 올려 둠.
- **예시**: 로그인 세션 확인. 대신 원본과 잠깐 어긋날 수 있어 **수명(TTL)** 을 정해 둔다.

## 2. Redis
- **정의**: **키-값** 방식의 **메모리 기반** 저장소(NoSQL의 일종). *Remote Dictionary Server*.
- **비유**: 계산대 옆 **초고속 메모판** — 번호(키)만 대면 값이 튀어나온다.
- **예시**: 세션 저장. 키마다 **자동 만료(TTL)** 를 걸 수 있어 "30분 활동 없으면 자동 로그아웃"을 쉽게 만든다.

## 3. SQL
- **정의**: 데이터베이스에 **말을 거는 언어**. 정해진 **형식(표 구조)에 맞는** 데이터를 다룬다(관계형).
- **비유**: 거의 **영어 문장** — `SELECT 이름 FROM 상품 WHERE 분류='케이크'` = "상품 표에서 케이크인 것의 이름을 줘".
- **예시**: SELECT(조회)·INSERT(생성)·UPDATE(수정)·DELETE(삭제) = **CRUD**.

## 4. NoSQL
- **정의**: 정해진 형식(고정된 표)에 **얽매이지 않는** 데이터를 다루는 DB (키-값·문서 등).
- **비유**: 칸이 고정된 엑셀 표 대신 **자유로운 메모장**.
- **예시**: **Redis**(키-값)가 대표적. "형식에 딱 맞지 않아도 되는 데이터"에 쓴다.

## 5. 색인(Index)
- **정의**: 자주 찾는 항목을 **미리 정리해 둬** 빠르게 찾게 하는 장치.
- **비유**: 두꺼운 책 뒤의 **'찾아보기'** ("삼투압 → 245쪽") — 처음부터 안 넘기고 바로 감.
- **예시**: 이메일·상품명에 색인 → 수백만 건도 순식간. 단 공짜는 아님(넣고 빼는 게 살짝 느려져서 아무 데나 안 건다).

## 6. Docker
- **정의**: 프로그램을 **'컨테이너'** 로 묶어, 명령 한 줄로 띄우고 지우는 도구.
- **비유**: 이사 짐을 **규격 컨테이너**에 담아, 어느 집에서든 그대로 푸는 것.
- **예시**: DB를 **내 컴퓨터를 어지럽히지 않고** 명령 한 줄로 켜고, 필요 없으면 통째로 지운다.

---

한 줄 정리: 오늘 용어는 **"데이터를 어디에 담고(SQL·NoSQL), 어떻게 빨리 찾고(색인·캐시·Redis), 어떻게 손쉽게 띄우는가(Docker)"** 로 이어진다.
