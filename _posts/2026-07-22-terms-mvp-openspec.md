오늘 배운 용어들을 **이해·암기용**으로 정리한다. "정의 + 비유 + 예시"로 짧게.

## 1. MVP(Minimum Viable Product, 최소 기능 버전)
- **정의**: 사용자가 **의미 있는 한 가지 일을 처음부터 끝까지** 해낼 수 있는, 의도적으로 작은 버전. "완성품 축소판"도 "화면만 그럴듯한 시제품"도 아니다.
- **비유**: **스케이트보드** — 느려도 실제로 굴러가 나를 데려다준다. (자동차 타이어 하나는 MVP가 아니다 — 그걸론 아무 데도 못 간다.)
- **예시**: 신청 앱이면 "로그인 → 클래스 골라 신청 → 내 신청 확인"이 **끝까지** 되는 것. 초라해도 끝까지 굴러가야 한다.

## 2. OpenSpec(오픈스펙)
- **정의**: 코드보다 **명세(스펙)를 먼저 확정**하고, 그 명세를 기준으로 구현·검증·기록을 관리하는 **스펙 주도** 도구. 일을 **change(변경 제안)** 단위로 다룬다.
- **비유**: 집 짓기 전 **설계도·시방서**를 먼저 그려 승인받고 착공하는 것.
- **예시**: Claude Code에서 `/opsx:explore → propose → apply → sync → archive` 순으로 흐름. "왜 이렇게 바꿨는지"가 변경 단위로 남는다. (실험적 워크플로라 명령은 바뀔 수 있음)

## 3. 퍼스트·세컨드·서드 파티 앱(1st / 2nd / 3rd-party app)
- **정의**: 클로드 앱을 **누가 만들었나**로 나눈 것. **퍼스트파티** = 앤트로픽이 직접 만듦(예: Claude 데스크탑 앱) / **세컨드파티** = 앤트로픽과 **파트너 관계** 회사가 만듦 / **서드파티** = 앤트로픽과 **무관한** 회사·개인이 만듦.
- **비유**: 게임기로 치면 **제조사 정품 게임(1st)** · **제휴사 게임(2nd)** · **아무 개발사나 만든 게임(3rd)**.
- **예시**: 내가 Agent SDK로 만든 클로드 앱 = **서드파티**. 셋은 신뢰·지원 수준이 다르다.

## 4. Agent SDK(에이전트 SDK)
- **정의**: 클로드를 이용해 **에이전트(스스로 도구를 쓰며 일하는 AI 프로그램)를 만들 수 있게** 해주는 개발 키트.
- **비유**: 로봇을 만들 때 쓰는 **부품 세트 + 설명서** — 바닥부터 안 만들고 이걸로 조립한다.
- **예시**: 이걸로 만든 내 클로드 앱은 위의 **서드파티** 앱이 된다.

## 5. 밴(Ban)
- **정의**: 정책·약관을 어겼을 때 서비스가 계정·접근을 **차단**하는 것. 보통 **안내 메일**로 사유가 통지된다.
- **비유**: 도서관에서 규칙을 어기면 **출입 정지** 통보를 받는 것.
- **예시**: 앱/API가 약관 위반(과도한 호출·악용 등) 시 밴. 안내 메일의 사유를 보고 소명·수정한다.

---

한 줄 정리: 오늘 용어는 **"작게 시작하고(MVP), 스펙부터 잡고(OpenSpec), 내가 만든 앱이 어디에 속하는지(파티 구분)와 무엇으로 만드는지(Agent SDK), 그리고 지켜야 할 선(밴)"** 으로 이어진다.
